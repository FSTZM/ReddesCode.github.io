Redis提升软件性能

需求分析

我们在添加shiro框架后，每次执行功能都会调用授权方法，而每次调用都会执行很多查询，这样会影响效率。怎么提升程序性能呢？我们可以考虑使用缓存数据库来实现。

Redis简介

Redis是一个高性能的key-value缓存系统。Redis的出现，很大程度补偿了memcached这类key/value存储的不足，在部分场合可以对关系数据库起来很好的补充作用。它提供了Java, C++, C#, PHP, JavaScript, Perl, Object-C, pathon, Ruby, Erlang等客户端，使用很方便

项目知识点总结

第一阶段 基础模块开发

项目第1天至第3天
1、Maven 项目的搭建

2、EasyUI 的常用的控件的属性、事件和方法

3、Hibernate 多对一关联

4、Struts2 属性驱动的应用

5、Spring 声明式事务

6、fastJSON 的使用

7、代码的抽取与生成

8、AJAX ( JQuery )与json的使用

目标:要求能够掌握表增删改查功能的编写,能够熟练运用弹出窗口、日期框、验证框等控件、能够理解和掌握前后端编程的开发技巧。

第二阶段 主线业务流程开发

项目第4天至第7天
1、登陆与主界面搭建

2、密码加密;

3、采购流程与相关业务逻辑

4、销售流程与相关业务逻辑

5、库存管理功能
目标:要求能够掌握EasyUI前端逻辑的编写, 能够掌握后端业务逻辑的编
写。

第三阶段 开发中的常用技术

项目第8天至第11天

1、运用HighCharts开发图表

2、使用javaMailSender发送邮件

3、使用Quartz框架实现任务调度

4、POI实现数据的导入导出

5、CXF 框架实现远程调用

6、多表关联实现统计报表

目标:我们能够在实际开发中运用上述技术解决实际中的特殊的业务需求。

第四阶段 安全、性能与团队开发

项目第12天至第14天

1、权限系统的搭建

2、Tree 控件的使用

3、多对多关联查询

4、Shiro 框架的认证与授权

5、Redis 与Jedis实现缓存数据

6、SVN在项目中的使用

目标:掌握权限系统的开发与Shiro框架、Redis 、SVN 等技术。

第五阶段 实战开发

目标:掌握团队开发的技巧,熟练使用SVN签入签出、手动Merge、沟通能力、自学能力、管理能力、团队开发经验、项目部署能力、语言表达能力、需求分析与设计能力。

表结构文档

1.退货订单表（RETURNORDERS）

字段名 类型(位数) 是否必填 说明
uuid number 必填 订单ID,主键
createtime date 创建日期
checktime date 审核日期
endtime date 结束日期
type char(1) 类型 1：采购退货 2：销售退货
creater number 创建人
checker number 审核人
ender number 结束人
supplieruuid number 供应商或客户ID
totalmoney number 总金额
state char(1) 状态 0:未审核 1:已审核 2：已结束
ordersuuid number 原订单ID

2.退货订单明细表（RETURNORDERDETAIL）

字段名 类型(位数) 是否必填 说明
uuid number 必填 主键
goodsuuid number 商品ID
goodsname varchar(50) 商品名称
price number(10,2) 价格
num number 数量
money number(10,2) 金额
endtime date 结束日期
ender number 结束人
storeuuid number 仓库ID
state char(1) 状态
ordersuuid number 退货订单ID

3.盘盈盘亏登记表（INVENTORY）

字段名 类型(位数) 是否必填 说明
uuid number 必填 主键
goodsuuid number 商品ID
storeuuid number 仓库ID
num number 盘盈（盘亏）数量
type char 类型 1：盘盈 2：盘亏
createtime date 登记日期
checktime date 审核日期
creater number 登记人
checker number 审核人
state char(1) 状态：0 未审核 1:已审核
remark varchar(200) 备注

评分标准

评分项 分数比重 备注

3.代码规范度（命名及结构） 10分
