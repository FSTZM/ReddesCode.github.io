DML简介

DML是对数据库表中的数据进行增、删、改的操作，即添加表中的数据，删除表中的数据，修改表中的数据。主要下面三个关键字：INSERT、DELETE、UPDATE。

MySQL workbench
MySQL workbench是MySQL官方发布的图形化界面工具，可以使开发者更便捷的使用MySQL操作数据库。
MySQL workbench下载：http://pan.baidu.com/s/1geDNrhx
除了这个之外还有一些其他关于操作数据库的图形化界面工具：
sqlyog
Navicat

INSERT
语法： INSERT INTO 表名（字段名1，字段名2 …） VALUES (值1，值2…);
注意：字段名与值的类型、个数、顺序要对应上。
值不要超出字段定义的长度，否则会报错。
如果插入空值，需要使用null
插入的日期和字符都使用单引号括起来。

上面语句中WHERE后面的是条件，即指出要修改哪一条数据，如果没有WHERE条件的话，会将表中所有的数据修改。

DELETE和TRUNCATE的区别：

DELETE只是删除表中的数据，不删除表，删除后的数据可以恢复。
TRUNCATE删除是把表直接DROP掉，然后再创建一个同样的新表。删除的数据不能找回。执行速度比DELETE快。

开发中通常使用DELETE。
