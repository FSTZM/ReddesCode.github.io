JDK的安装和环境变量设置
发表于 2019-09-13| 更新于 2020-04-26|IT后端JavaJavaSEJava基础教程第一章 Java概述
阅读量:

JDK的下载和安装

可以去oracle官网里面下载JDK：http://www.oracle.com
英文不好的同学可以通过网站中java工具栏目中下载

安装注意事项：

  • 安装路径中最好不要出现中文、空格或特殊字符
  • 最好不要安装到C盘，我的安装路径：D:\Java\jdk1.8.0_60
  • 当提示安装JRE时可以选择不安装，因为JDK里面包含JRE了。

常用的dos命令

安装好jdk之后，我们主要使用里面的的命令，这些命令可以从命令提示符中输入。快捷键win+d，输入cmd，打开命令提示符。

  • dir 显示当前目录中的子目录和文件
  • cd 切换当前目录
  • cls 清屏
  • tab 自动补全
  • ipconfig 显示网络配置信息
  • 通过向上/向下的箭头， 查看之前使用过的命令

配置环境变量

为了更方便以后执行Java相关命令，需要配置环境变量。右键我的电脑—属性—高级系统设置—环境变量。!

  1. 在系统变量里面，点击新建，
    变量名填写JAVA_HOME
    变量值填写D:\Java\jdk1.8.0_60
    变量值填写你本地的JDK安装的路径即可，我本地的安装路径是D:\Java\jdk1.8.0_60。

  2. 在系统变量里面，先找下变量名为PATH的选项（如果没有则点击新建，变量名填写PATH），在PATH里面的变量值的最前面加上%JAVA_HOME%\bin;
    注意不要少了分号；

环境变量配置完成之后，打开命令提示符，快捷键win+r，输入cmd。
在命令提示符里面输入java和javac两个命令，如果出现一堆类似图片中的文字则说明配置成功。

classpath，在JDK5版本之后可以不用配置classpath了，如果你一定要配置的话，可以将其配置为.（即一个点，表示当前路径）。

JDK安装路径下的目录介绍

安装完JDK后，JDK的安装路径下会有一些文件夹，那这些文件夹里面的文件都是做什么的呢？

  • bin目录：该目录用于存放一些可执行程序。
  • include目录：由于JDK是通过C和C++实现的，因此在启动时需要引入一些C语言的头文件，该目录就是用于存放这些头文件的。
  • lib目录：lib是library的缩写，里面存放了 Java 类库或库文件，是开发工具使用的归档包文件。
  • src.zip文件：src.zip为src文件夹的压缩文件，src中放置的是JDK核心类的源代码，通过该文件可以查看Java基础类的源代码。
